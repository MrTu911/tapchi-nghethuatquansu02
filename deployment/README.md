## 🖥️ Yêu Cầu Hệ Thống

### Phần cứng tối thiểu
- **CPU:** 2 cores
- **RAM:** 4GB (khuyến nghị 8GB)
- **Ổ cứng:** 50GB SSD
- **Mạng:** 100Mbps

### Phần mềm
- **Hệ điều hành:** Ubuntu 20.04 LTS / 22.04 LTS
- **Node.js:** >= 18.x
- **PostgreSQL:** >= 14.x
- **Nginx:** >= 1.18 (cho reverse proxy)
- **Yarn:** Latest

### Mạng
- **Port mở:** 22 (SSH), 80 (HTTP), 443 (HTTPS)
- **Port ứng dụng:** 3000 (mặc định, có thể thay đổi)

## 💾 Backup và Restore

### Backup Tự Động

Script tự động backup mỗi ngày lúc 2:00 AM:
- Database: `db_YYYYMMDD_HHMMSS.sql.gz`
- Uploads: `uploads_YYYYMMDD_HHMMSS.tar.gz`
- Lưu trữ: 30 ngày

### Backup Thủ Công

```bash
# Chạy script backup
sudo /opt/tapchi-hcqs/scripts/auto-backup.sh

# Kiểm tra backups
ls -lh /opt/tapchi-hcqs/backups/
```

### Restore Từ Backup

#### 1. Dừng ứng dụng
```bash
sudo systemctl stop tapchi-hcqs
```

#### 2. Restore Database
```bash
cd /opt/tapchi-hcqs/backups

# Giải nén
gunzip db_20240108_020000.sql.gz

# Restore
sudo -u postgres psql tapchi_hcqs < db_20240108_020000.sql
```

#### 3. Restore Uploads
```bash
cd /opt/tapchi-hcqs/backups

# Giải nén và restore
tar -xzf uploads_20240108_020000.tar.gz -C /opt/tapchi-hcqs/

# Fix permissions
sudo chown -R tapchi:tapchi /opt/tapchi-hcqs/uploads
```

#### 4. Khời động lại
```bash
sudo systemctl start tapchi-hcqs
```

### Backup Toàn Bộ Hệ Thống

```bash
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/backup/tapchi"

mkdir -p $BACKUP_DIR

# Backup database
sudo -u postgres pg_dump tapchi_hcqs | gzip > $BACKUP_DIR/db_$DATE.sql.gz

# Backup application
sudo tar -czf $BACKUP_DIR/app_$DATE.tar.gz \
  --exclude='node_modules' --exclude='.next' --exclude='.build' \
  /opt/tapchi-hcqs

# Backup nginx config
sudo cp /etc/nginx/sites-available/tapchi-hcqs $BACKUP_DIR/nginx_$DATE.conf

# Backup systemd service
sudo cp /etc/systemd/system/tapchi-hcqs.service $BACKUP_DIR/service_$DATE

echo "Backup completed: $BACKUP_DIR"
```

## ⚡ Xử Lý Sự Cố

### Ứng dụng không khởi động

```bash
# Xem logs chi tiết
sudo journalctl -u tapchi-hcqs -n 100 --no-pager

# Kiểm tra port có bị chiếm không
sudo netstat -tulpn | grep 3000

# Kiểm tra database connection
sudo -u postgres psql -c "SELECT 1;"

# Kiểm tra .env file
sudo cat /opt/tapchi-hcqs/.env

# Thử chạy manual để xem lỗi
cd /opt/tapchi-hcqs
sudo -u tapchi yarn start
```

### Database connection error

```bash
# Kiểm tra PostgreSQL đang chạy
sudo systemctl status postgresql

# Khởi động PostgreSQL
sudo systemctl start postgresql

# Kiểm tra user/database
sudo -u postgres psql -c "\du"
sudo -u postgres psql -c "\l"

# Test connection
sudo -u postgres psql -U tapchi_user -d tapchi_hcqs -c "SELECT 1;"
```

### Build errors

```bash
# Xóa cache và build lại
cd /opt/tapchi-hcqs
sudo -u tapchi rm -rf .next .build node_modules
sudo -u tapchi yarn install
sudo -u tapchi yarn prisma generate
sudo -u tapchi yarn build
```

### Out of memory

```bash
# Tăng memory limit khi build
cd /opt/tapchi-hcqs
sudo -u tapchi NODE_OPTIONS="--max-old-space-size=4096" yarn build

# Hoặc tạo swap file
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

### Permission errors

```bash
# Fix ownership
sudo chown -R tapchi:tapchi /opt/tapchi-hcqs

# Fix permissions
sudo chmod -R 755 /opt/tapchi-hcqs
sudo chmod 600 /opt/tapchi-hcqs/.env
sudo chmod -R 755 /opt/tapchi-hcqs/uploads
```

### Nginx errors

```bash
# Test nginx config
sudo nginx -t

# Xem logs
sudo tail -f /var/log/nginx/error.log

# Restart nginx
sudo systemctl restart nginx
```

### Disk full

```bash
# Kiểm tra dung lượng
df -h

# Xem thư mục lớn
du -sh /opt/tapchi-hcqs/*

# Dọn logs cũ
sudo journalctl --vacuum-time=7d

# Dọn backups cũ
find /opt/tapchi-hcqs/backups -mtime +30 -delete
```

## 🔒 Bảo Mật

### Firewall

```bash
# Xem rules hiện tại
sudo ufw status

# Chặn IP
sudo ufw deny from <IP-ADDRESS>

# Mở port
sudo ufw allow 3000/tcp
```

### SSL Certificate (Let's Encrypt)

```bash
# Cài đặt Certbot
sudo apt-get install certbot python3-certbot-nginx

# Lấy certificate
sudo certbot --nginx -d your-domain.com

# Auto-renew
sudo certbot renew --dry-run
```

### Fail2ban

```bash
# Cài đặt
sudo apt-get install fail2ban

# Cấu hình cho Nginx
sudo nano /etc/fail2ban/jail.local
```

Thêm:
```ini
[nginx-limit-req]
enabled = true
filter = nginx-limit-req
action = iptables-multiport[name=ReqLimit, port="http,https", protocol=tcp]
logpath = /var/log/nginx/error.log
findtime = 600
bantime = 7200
maxretry = 10
```
